# 用語集

本書に登場した主な用語を、初めて読む人向けに簡潔にまとめます。詳しい説明は各章を参照してください。章をまたいで何度も顔を出す言葉を中心に選んでいます。

## 全体に関わる用語

**意味解析（semantic analysis）**
: AST を入力に取り、プログラムが意味として筋が通っているかを調べ、実行や最適化に必要な情報を導く工程。構文解析の次に来る。

**抽象構文木（AST, abstract syntax tree）**
: プログラムの構文構造を表す木。本書の出発点。各ノードに解析結果（型やスコープ情報）を書き込むことで「装飾」される。

**静的解析（static analysis）／動的解析（dynamic analysis）**
: プログラムを実行せずに（ソース＝AST から）調べるのが静的解析、実際に実行しながら調べるのが動的解析。本書は主に静的解析を扱う。

**健全（sound）／完全（complete）**
: 「問題なし」と言ったら本当に問題ないのが健全。「問題あり」を見逃さないのが完全。一般に両立できず、どちらに倒すかが設計判断になる。

**偽陽性（false positive）／偽陰性（false negative）**
: 本当は問題ないのに「あやしい」と誤報するのが偽陽性、本当は問題あるのに見逃すのが偽陰性。

**判定不能（undecidable）**
: どんなアルゴリズムでも有限時間で必ず正しく答えを出すことができない問題の性質。プログラムの非自明な性質はライスの定理により一般に判定不能で、静的解析が近似に頼る根本理由。

**属性文法（attribute grammar）**
: 文法記号に属性（値）を結びつけ、その計算規則を記述する枠組み。情報が下から上へ流れる**合成属性**と、上から下へ流れる**継承属性**がある。

## 名前と型

**名前解決（name resolution）**
: 名前の各使用を、その宣言に結びつける解析。すべての意味解析の土台。

**スコープ（scope）**
: ある宣言が参照できるプログラム上の範囲。入れ子になり、内側から外側へたどって名前を探す（スコープチェーン）。

**シンボルテーブル（symbol table）**
: 名前からその情報（種類・型・宣言位置など）への対応表。スコープごとに入れ子で管理する。

**シャドーイング（shadowing）**
: 内側のスコープの宣言が、外側の同名の宣言を隠すこと。

**型（type）／型システム（type system）**
: 型は値の「種類」を表すラベル。型システムは、どの式にどんな型が付くかを定める規則（型規則）の全体。

**型環境（type environment, Γ）**
: いま見えている変数とその型の対応表。型検査・型推論で持ち回る。

**型の健全性（type soundness）**
: 「型の付いたプログラムは型エラーで止まらない」という保証。進行（progress）と保存（preservation）の2補題で証明される。

**サブタイピング（subtyping）**
: 「`S` は `T` の一種だから、`T` が要求される場所に `S` を置ける」という型の包含関係（`S <: T`）。

**型推論（type inference）**
: 型注釈なしのプログラムから型を自動的に導く解析。代表がヒンドリー・ミルナー（HM）法。

**型変数（type variable）／単一化（unification）**
: 未知の型を表すプレースホルダが型変数。2つの型を等しくするための型変数への当てはめ（代入）を求める操作が単一化。

**主要型（principal type）／let 多相**
: 型が付くなら存在する「最も一般的な型」が主要型。`let` 束縛の型を一般化して、使うたびに別の型として使えるようにするのが let 多相。

## フロー解析と抽象解釈

**制御フローグラフ（CFG, control flow graph）**
: 基本ブロックを節点、実行が移りうる方向を辺とする有向グラフ。フロー解析の舞台。

**基本ブロック（basic block）**
: 途中で枝分かれも飛び込みもない、一直線に実行される文の並び。

**支配関係（dominance）**
: あるブロックを必ず通らないと別のブロックに到達できない、という関係。ループ検出や SSA 構築に使う。

**データフロー解析（data flow analysis）**
: CFG の各地点でのある種の情報（生存変数・到達定義など）を、連立方程式の不動点として求める解析。

**伝達関数（transfer function）／合流演算（meet/join）**
: ブロックを通ると情報がどう変わるかが伝達関数、複数の経路が合流する点で情報をどう束ねるかが合流演算。

**不動点反復（fixpoint iteration）**
: 「変化がなくなるまで方程式を適用し続ける」計算法。格子の単調性と有限性から必ず収束する。ワークリストで効率化する。

**格子（lattice）**
: 要素間に精密／粗いの順序が入り、任意の2要素に上限が存在する集合。データフロー値・抽象値の住む数学的舞台。

**SSA 形式（static single assignment）**
: 各変数への代入をただ1か所にした中間表現。合流点を φ関数で表す。最適化を簡単・高速にする。

**指示解析（points-to / pointer analysis）**
: 各ポインタ・参照が指しうる対象の集合を求める解析。ポインタのある言語で他の解析の精度を支える。

**抽象解釈（abstract interpretation）**
: 値の抽象的な性質のレベルでプログラムを一度だけ実行し、すべての具体的実行を健全に覆う静的解析の統一理論。

**抽象領域（abstract domain）／ガロア接続**
: 性質を表す近似された値の世界が抽象領域（符号領域・区間領域など）。具体と抽象を安全な近似として結ぶ対応がガロア接続。

**ウィドニング（widening）**
: 無限に高い格子で不動点反復を有限時間で止めるため、わざと粗く近似して収束を強制する操作。

## 最先端の手法

**篩型（refinement type）**
: 型に「正である」「ゼロでない」などの述語を載せて表現力を高めた型。検査には SMT ソルバを使う。

**SMT ソルバ（SMT solver）**
: 整数の不等式や配列などの理論を組み込み、論理式が成り立つかを自動判定するエンジン。Z3 が代表。

**エフェクトシステム（effect system）**
: 式が返す値の型に加えて、実行で起きる副作用（IO・例外など）も型で追跡する枠組み。

**リージョン（region）／所有権（ownership）／借用（borrowing）**
: 値の寿命を静的に追ってメモリ安全を保証する仕組み。リージョンは確保・解放の単位、所有権は「持ち主は1つ」、借用は参照の貸し出し規則。Rust の中核。

**分離論理（separation logic）**
: 分離積 `P * Q`（メモリを重ならない2領域に分ける）によって、ポインタ操作を局所的に推論できるようにした論理。
